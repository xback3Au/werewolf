用户询问如何设计数据结构来构建RAG（检索增强生成）系统。这是一个关键问题，需要仔细设计。

用户的目标是：

让AI理解玩家发言和投票的深层逻辑
能评价操作好坏
能回答"狼人怎么找到神的"这类问题
为了支持这些能力，RAG需要保存：

时间线信息（什么时候发生了什么）
因果关系（A导致B相信C是狼）
多视角（不同玩家看到的信息不同）
操作评价（好坏的标准）
让我设计一个适合狼人杀对局分析的RAG数据结构。

这是一个关键问题。让我设计一套适合狼人杀对局分析的 RAG 数据架构。

核心设计原则

1. 时间线是一切的基础（狼人杀是时序游戏）
2. 因果关系比事实更重要（为什么要这样做）
3. 多视角是理解的关键（他知道什么，他不知道什么）
4. 可追溯性（能回答"为什么"）
推荐的表结构设计
表1: 时间线事件 (timeline_events)

CREATE TABLE timeline_events (
    id INTEGER PRIMARY KEY,
    analysis_id TEXT,
    day INTEGER,           -- 第几天
    phase TEXT,            -- 阶段: night/day/sunset/vote
    event_type TEXT,       -- 类型: speech/vote/death/badge_transfer
    event_id TEXT,         -- 关联ID (如 speech_id, vote_id)
    
    -- 事件内容摘要
    content TEXT,          -- "8号起跳通灵师，给5号发平民"
    
    -- 公开性（谁能看到这个事件）
    is_public BOOLEAN,     -- 所有人可见？
    visible_to TEXT,       -- JSON: ["1","2","3"] 或 "all"
    
    created_at TEXT
);

表3: 推理链 (inference_chains) ⭐ 核心表

CREATE TABLE inference_chains (
    id INTEGER PRIMARY KEY,
    analysis_id TEXT,
    
    -- 推理者信息
    observer_id TEXT,      -- 谁在做推理（如"3号玩家"）
    observer_role TEXT,    -- 他的真实身份（狼人/平民等）
    observer_perspective TEXT, -- 他当时知道什么（JSON）
    
    -- 推理对象
    target_id TEXT,        -- 推理谁（如"8号玩家"）
    target_day INTEGER,    -- 第几天的判断
    
    -- 推理内容
    conclusion_type TEXT,  -- "identity" / "intent" / "relationship"
    conclusion TEXT,       -- "8号是通灵师"
    confidence REAL,       -- 置信度 0-1
    reasoning TEXT,        -- 推理过程
    
    -- 证据链（关键！）
    evidence_ids TEXT,     -- JSON: ["speech_12", "vote_8"]
    evidence_summary TEXT, -- "基于8号的心路历程和票型"
    
    -- 正确性验证（用于后续学习）
    is_correct BOOLEAN,    -- 是否正确
    truth TEXT,            -- 真实身份/意图
    error_type TEXT,       -- 错误类型（如"被误导"、"信息不足"）
    
    -- 时间戳
    created_at TEXT
);
表4: 知识片段 (knowledge_chunks) ⭐ RAG核心

CREATE TABLE knowledge_chunks (
    id INTEGER PRIMARY KEY,
    analysis_id TEXT,
    
    -- 片段类型
    chunk_type TEXT,       -- "strategy" / "mistake" / "insight" / "relationship"
    
    -- 内容
    content TEXT,          -- 自然语言描述
    embedding TEXT,        -- 向量嵌入（用于语义检索）
    
    -- 上下文
    game_context TEXT,     -- 对局背景（版型、天数）
    related_players TEXT,  -- JSON: ["3","8"] 涉及的玩家
    related_events TEXT,   -- JSON: ["speech_15", "vote_day2"]
    
    -- 元数据
    importance_score REAL, -- 重要性
    is_teaching_material BOOLEAN, -- 是否可作为教学案例
    
    created_at TEXT
);
表5: 操作策略 (operation_strategies)

CREATE TABLE operation_strategies (
    id INTEGER PRIMARY KEY,
    analysis_id TEXT,
    
    -- 操作者
    player_id TEXT,
    player_role TEXT,      -- 真实身份
    
    -- 操作内容
    operation_type TEXT,   -- "vote" / "speech" / "kill" / "save" / "check"
    day INTEGER,
    
    -- 评价
    effectiveness_score REAL,  -- 有效性评分
    skill_level TEXT,      -- "beginner" / "intermediate" / "advanced" / "expert"
    
    -- 分析
    good_points TEXT,      -- 做得好的地方
    bad_points TEXT,       -- 可以改进的地方
    alternative_actions TEXT, -- 其他可能的选择
    
    -- 教学价值
    lesson_learned TEXT,   -- 可以学到什么
    applicable_scenarios TEXT, -- 适用于什么场景
    
    created_at TEXT
);
